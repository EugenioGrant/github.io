Democratizando Machine Learning con R Shiny

Eugenio Grant

2021-01-25

Antecedentes

Para muchas empresas la utilizacin de modelos de Machine Learning es un elemento fundamental de ventaja competitiva, por esto se vuelve especialmente importante encontrar formas de poner a disposicin de sus usuarios la posibilidad de usar estas herramientas.

Sin embargo el proceso de democratizar estos modelos enfrenta importantes barreras; su la complejidad implica que en muchos casos se requiere de usuarios con un nivel importante de conocimiento tcnico y por otro lado el costo de ponerlos a disposicin de todos los usuarios implica costos de desarrollo y el pago de plataformas especializadas.

R & R Shiny

RShiny es una herramienta poderosa para generar productos de datos desde R, ya sean visualzaciones o modelos predictivos. Este paquete tiene una excelente integracin con RStudio lo que hace que sea relativamente sencillo crear una aplicacin y ejecutarla localmente. Igualmente R Studio ofrece alternativa de pago para despligue de aplicaciones en la nube. Cada una de estas alternativas tiene sus ventajas y desventajas.

Alternativas de Despliegue (Deployment)

1. shinyapps.io: Servicio manejado por RStudio que permite publicacin de Apps con un manejo transparente del proceso de DevOps con solo unos cuantos clicks. Servicio limitado en trminos de recursos.

2. Shiny Server Open Source: Plataforma para generar un servidor web dedicado para el hosting de aplicaciones en R Shiny (on-premise). Slo soporta un poceso de R por App.

3. RStudio Connect: Plataforma de publicacin, producto comercial de RStudio y con muchisimas funcionalidades. Costosa en escenarios de prototipado.

4. Shinyproxy: Desarrollo Open-Source. Java en el lado de servidor y utiliza docker. Es una herramienta poderosa con caractersticas empresariales pero Open Source.

Veredcto

RStudio Connect es la alternativa ms completa en trminos de funcionalidad y simplificacin de los procesos de desarrollo, pero con licenciamiento de ~USD$10.000/anual se vuelve una barrera, ms an si estamos empezando a prototipar una herramienta especfica. Ante esto propongo un Stack Open Source

Docker empaqueta software en unidades estandarizadas llamadas contenedores que incluyen todo lo necesario para que el software se ejecute, incluidas bibliotecas, herramientas de sistema y cdigo.
